### 切块规则

##### 知识库字段标准

| 列名              | 是否必须 | 为什么必须                                                   |
| ----------------- | -------- | ------------------------------------------------------------ |
| `chunk_id`        | 必须     | 每个知识块唯一编号，后续入库、纠错、引用都靠它               |
| `book_title`      | 必须     | 多本书合库后必须区分来源                                     |
| `chapter_title`   | 必须     | 教材是强结构文档，章信息对检索很重要                         |
| `section_title`   | 必须     | 比章更细，实际检索非常常用                                   |
| `page_start`      | 必须     | 做引用追溯必须有起始页                                       |
| `page_end`        | 建议保留 | 跨页内容常见，保留更稳                                       |
| `chunk_title`     | 必须     | 相当于这块内容的人工摘要标题，提升可读性和后续人工检查效率   |
| `topic_type`      | 必须     | 后续可做过滤检索，如“只找症状/防治/病原”                     |
| `cleaned_text`    | 必须     | 真正要进入知识库和向量库的正文                               |
| `keywords`        | 必须     | 对农业/医学这类专业语料，术语词很关键，能辅助检索和后续查询扩展 |
| `citation_anchor` | 必须     | 直接用于可追溯引用，后面做带引用的问答评测很重要                   |

##### 标签说明

| 标签名            | 标签含义                |
| ----------------- | ----------------------- |
| `definition`      | 定义类知识              |
| `pathogen`        | 病原体类知识            |
| `symptom`         | 症状类知识              |
| `occurrence_rule` | 发生规则/流行规律类知识 |
| `transmission`    | 传播途径类知识          |
| `diagnosis`       | 诊断依据类知识          |
| `control_method`  | 控制方法/防治措施类知识 |
| `summary`         | 概括、总结类知识        |

##### 切块标准

+ 一个知识块只讲一个核心主题。
2. 优先按知识完整性切块，而不是机械按字数切块。
3. 具体病害内容可精细切块，如病原、症状、发生规律、防治分别成块。
4. 概说、流行和预测、一般防治原则等内容可粗粒度切块，但不能完全删除。
5. 清洗后正文应保持原意，不得自行发挥或补充书中未出现的结论。
6. 每个知识块必须填写页码、关键词和引用锚点。
7. 关键词使用分号分隔，建议 3～8 个。
8. 内容类型只能从统一 topic_type 标签中选择。
9. 扫描件错误较多时，以原 PDF 为准，MinerU 结果仅作结构辅助。
10. 同一本书内字段填写格式必须统一。
